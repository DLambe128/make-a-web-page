Lesson 1: The Basics of the Web and HTML

Introduction

The “world wide web” is a collection of HyperText Markup Language (HTML) documents. Although web pages can can contain many types of files (such as plain text, HTML, images, videos, and music), HTML is the main component of most web pages. The links between web pages are called hyperlinks.

Your browser (a program that runs on your computer, such as IE and Firefox) makes HTTP-protocol requests via to the internet (the world’s largest computer network) to servers (computers that host the files that make up the web), and then the servers respond with files (HTML, videos, images, etc.) that your browser then displays.

This video further explains the concepts I have discussed.

More on HTML

HTML consists of text content (what the user sees), markup (what it looks like), references to other documents (such as images and videos), and links to other pages.

This line is just plain text. You can recognize markup in an HTML document by the tags that surround it. In HTML, the entire unit containing a tag, its associated closing tag, and the content within the tag pair is called an element.

Tags!

The bold tag <b> ... </b> makes text bolded, whereas the em tag <em> ... </em> makes text italicized.If you don't include a closing tag for the 'em' tag, everything after the opening tag will be italicized (until you put in a closing tag).

Certain tags, like the anchor (or just 'a' tag) can contain one or more attributes (like href). Attributes are set equal to a value, which is enclosed in quotation marks. Anchor tags are used specifically for creating links to other files. This link back to the udacity web-site was created with an anchor tag pair with an 'href' attribute, like so:

<a href="Website URL">Display Text</a>

The 'img' tag is used to link out to an image. The image tag usually contains 'src' and and 'alt' attributes. The 'alt' attribute value is what a user will see if the image (set equal to the 'src' attribute) doesn't load. Unlike other tags, the 'img' tag does not use a closing tag! Lone tags (that don't have a closing tag) are called void tags.

Creating space in HTML

In HTML, white space (new lines, tabs, spaces) in your HTML document aren't visible in the browser. To force HTML to recognize a new line on your page (sort of like hitting the Enter key on Word), you have to put in a br, or break tag: <br> As you can see, the br tag is a void tag.

The p, or paragraph, tag can be used to create an invisible box (with height and width) around its contents: <p>**content**</p>

Inline vs. Block

Inline elements: 'b', 'em', 'br', 'img', 'a', 'strong' and 'span'
Block elements: 'p', 'div' and 'form'

Lesson 3: Adding CSS Style and HTML Structure

More on CSS

CSS, again, determines the visual structure of the page. With CSS, the most specific rule is applied to every element ("Cascading"). This is also called inheritance, which you can read more about here. Understanding inheritance requires abstract thinking. It is important to note that changing the visual elements of a page only changes how the element is displayed, and does not modify the element itself.

Selectors

CSS can not only modify classes (".pageheader"), but also repeated elements on a page (like p, h1, etc.) - these are called selectors. When we say something like "padding:20px;", padding is the property and 20px is the value. You can read more about selectors, including combined selectors, here. Check out Andy Brown's CSS Summary and Reference, which includes a section on how to include CSS styling in a web page:

  • Write CSS in the <head>of the HTML doc (with a<style>element) - use for smaller projects
  • Link your HTML to a separate CSS file (with a <link> tag inside the head tag, with distinct HTML and CSS files). NOTE: For this method, make sure the HTML and CSS file are in the same folder.
  • Write your style inline with the HTML - generally not recommended, because leads to a lot of repeated code. e.g., <div style="background-color: red; color: white">

Using Semantic Tags

Make sure to use the right type of element for certain parts of the page, depending on what the purpose of the element is. For example, use header elements for title sections. Search engines use headers to index the structure and content of a web page. Note that using an h1 HTML tag around text, the text increases in size and a box is created around the element (extra space above and below text).

More on the Box Model

Boxes on web pages consists of content, padding, border and margins (the latter 3 must be added with CSS properties). The Computed tab in developer tools will show all of these and their sizes.

The content is the image or text that appears on the web site, the "nucleus". The padding surrounds the content, and functions to clear an area between the content and border; it inherits the background color of the box. The border surrounds the content and padding, inheriting the color property of the box. The margin, which doesn't have a background color (transparent), clears an area between that element and the next element.

By default, the size of the element itself is equal to the sum of the border, padding and content widths (margin not included). To avoid having to re-calculate the actual width of the element after changing any of these values (border, padding, or content), use the CSS property "box-sizing:border-box;". This makes the size of the element include both the border and padding - you set the width of the overall box, and the size of the box always stays the same. This property has been developed recently, so you must include browser-specific prefixes so older browsers can display it.

You can also set box sizes with percentages, as opposed to pixels (box sizes stay that that pixel size, regardless of page size); in this case, size will change depending on how large the screen is. The max width property determines the maximum width a box can take-up as a browser window is made larger.

Box Positioning

In order to have boxes appear next to each other (as opposed to stacked on top of one another), set display:flex; within the parent element. The size of the boxes must be less than 100%. You can read more about the wonder that is flex box here. By default, divs are block elements (as opposed to inline), and therefore they take up the entire width of a page; display:flex; overrides this behavior.
